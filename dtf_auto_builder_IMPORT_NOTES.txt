DTF Auto Builder — Medusa product import CSV
==============================================

File: dtf_auto_builder_medusa_import.csv

This matches the column layout used in this repo (see syzmik_medusa_import_part_01.csv and
scripts/verify_medusa_product_import_csv.js). The file has been run through
`node scripts/verify_medusa_product_import_csv.js dtf_auto_builder_medusa_import.csv` successfully.

BEFORE you import
-----------------
1) Replace every YOUR_SALES_CHANNEL_ID with your real sales channel id from
   Medusa Admin → Settings → Sales channels → open the channel your store uses → copy id
   (format like sc_01KP...). This must be the same channel linked to your publishable API key.

2) **Product Id (first column):** Must be set on **every** row for an **update** when the product
   already exists. The CSV uses `prod_01KQ1B4VVD1TBNKB5GY5Z1SK95` from your Admin product URL; if
   your id differs, replace it on all 5 data rows (or run:
   `python3 scripts/fill_product_ids_in_import_csv.py` with your env file — see that script).
   If Product Id is **blank**, Medusa **creates** a new product and errors with
   "Product with handle: dtf-auto-builder, already exists."

3) Prices in "Variant Price AUD" are in dollars (e.g. 24.00 = A$24.00), matching the seed
   amounts: $24, $48, $72, $96, $120 for the five lengths.

4) If the product dtf-auto-builder does not exist yet, **clear** the Product Id on all five rows
   (empty first column) so the import **creates** the product, or use a new handle.

5) If your CSV import expects EUR/USD as well, add price columns the same way your other import
   files do; this file only sets AUD to match a typical AU storefront.

6) After import, remove any leftover "default" option/variant if Admin shows duplicates.

Gang sheet size values use the Unicode multiply sign (×) as in the storefront parser.
